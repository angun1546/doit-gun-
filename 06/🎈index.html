HTML문서는 웹페이지의 뼈대 역할을 함
계층구조를 가진다

태그의 사용법

태그는 여는태그와 닫는태그로 구성됨

태그를 표시하는 기호는 < > 이며 괄호내에 명령어를 넣어 사용한다. 공백을 표시할때는            를 사용한다.

태그의 분류

태그가 웹페이지에서 차지하는 영역에 따라 두가지 나눌수 있다.

분류 특징 주요 태그
블록 레벨 (Block Level) 웹페이지의 화면 전체 너비를 차지
줄바꿈이 자동으로 일어남		 <div>, <p>, <h1>~<h6>, <ul>, <ol>, <li>, <table>, <form>, <header>, <footer>, <section>
인라인 레벨 (Inline Level) 콘텐츠의 너비만큼만 차지
줄바꿈이 일어나지 않음		 <span>, <a>, <img>, <strong>, <em>, <input>, <label>, <br>, <button>

div와 span 상세 설명

구분 div span
타입 블록 레벨 (Block Level) 인라인 레벨 (Inline Level)
화면 차지 웹페이지의 전체 너비를 차지 콘텐츠의 너비만큼만 차지
줄바꿈 줄바꿈이 자동으로 일어남 줄바꿈이 일어나지 않음
용도 큰 영역을 나누어 구조화 텍스트 일부를 강조하거나 스타일링

div 설명

div는 Division의 약자로, 웹페이지를 여러 섹션으로 나누는 데 사용하는 블록 레벨 태그입니다.

특징:

  • 전체 너비를 차지하므로 자동으로 줄바꿈
  • CSS로 크기, 배경색, 여백 등을 자유롭게 조절 가능
  • 웹페이지의 전체 구조를 만드는 데 사용

이것은 첫 번째 div입니다. 전체 너비를 차지합니다.
이것은 두 번째 div입니다. 자동으로 새 줄에서 시작합니다.

span 설명

span은 텍스트나 콘텐츠의 일부에 스타일을 적용하거나 강조할 때 사용하는 인라인 레벨 태그입니다.

특징:

  • 콘텐츠의 너비만큼만 차지
  • 같은 줄에 여러 개의 span을 나열할 수 있음
  • 텍스트의 특정 부분을 강조하거나 색상을 변경할 때 사용

span 시각적 예시:

HTML은 웹페이지의 구조를 만들고, CSS는 스타일을 적용합니다.

div vs span 비교

div 사용 (블록 레벨):

첫 번째 div
두 번째 div - 새 줄에서 시작

span 사용 (인라인 레벨):

첫 번째 span 두 번째 span - 같은 줄에 표시 세 번째 span - 계속 같은 줄

언제 어떤 것을 사용할까?

  • div 사용: 헤더, 네비게이션, 콘텐츠 영역, 푸터 등 큰 레이아웃 구성
  • span 사용: 텍스트 일부를 빨간색으로 표시, 특정 단어 강조, 배지나 라벨 표시

블록과 인라인 사용 시 주의사항

블록 레벨 태그 사용 시 주의점

❌ 주의 1: 블록 태그 안에 블록 태그

문제: div나 p 안에 다른 블록 태그를 넣을 때 구조를 체계적으로 관리해야 합니다.

나쁜 예시:

<p> <div>이렇게 하면 안 됩니다</div> </p>

좋은 예시:

<div> <p>이렇게 사용하세요</p> </div>

❌ 주의 2: 지나친 div 중첩

문제: div를 너무 많이 중첩하면 코드가 복잡해지고 유지보수가 어려워집니다.

나쁜 예시:

<div> <div> <div> <div>내용</div> </div> </div> </div>

좋은 예시:

<div class="container"> <p>내용</p> </div>

❌ 주의 3: 블록 태그의 높이 설정

문제: 블록 태그는 기본적으로 콘텐츠의 높이만큼만 차지합니다. 높이를 명시적으로 설정하지 않으면 예상과 다를 수 있습니다.

나쁜 예시:

<div> 전체 높이를 채우고 싶지만 설정하지 않음 </div>

좋은 예시:

<div style="height: 200px; background: #eee;"> 높이를 설정하여 명확하게 표시됨 </div>

인라인 레벨 태그 사용 시 주의점

❌ 주의 1: 인라인 태그에 너비와 높이 설정 불가

문제: 인라인 태그(span, strong, em 등)에는 width, height가 적용되지 않습니다.

나쁜 예시:

<span style="width: 200px; height: 100px;"> 너비와 높이가 적용되지 않음 </span>

해결 방법: CSS에서 display 속성을 변경해야 합니다.

<span style="display: block; width: 200px; height: 100px;"> 이제 너비와 높이가 적용됨 </span>

❌ 주의 2: 인라인 태그 안에 블록 태그

문제: span, strong, em 등 인라인 태그 안에 div, p 같은 블록 태그를 넣으면 안 됩니다.

나쁜 예시:

<span> <div>이렇게 하면 안 됩니다</div> </span>

좋은 예시:

<div> <span>이렇게 사용하세요</span> </div>

❌ 주의 3: 인라인 태그의 여백 설정

문제: 인라인 태그는 margin-top, margin-bottom이 적용되지 않습니다.

나쁜 예시:

<span style="margin: 20px;"> 위아래 여백이 적용되지 않음 </span>

해결 방법: padding을 사용하거나 display를 변경합니다.

<span style="padding: 10px; background: #eee;"> 패딩으로 내부 여백 적용 </span>

일반적인 주의사항

⚠️ 주의 1: 의미있는 태그 선택

모든 것을 div로만 만들지 마세요. 제목은 h1~h6, 강조는 strong, em을 사용하세요.

❌ <div style="font-size: 32px; font-weight: bold;">제목</div> ✓ <h1>제목</h1>

⚠️ 주의 2: 인라인-블록 (Inline-Block)

인라인처럼 배치하면서 블록의 속성을 원할 때는 CSS display 속성을 사용합니다.

<span style="display: inline-block; width: 100px; height: 50px;"> 이제 너비와 높이가 적용됩니다 </span>

✓ 좋은 습관

  • 큰 영역을 나눌 때: div, section, header, footer 사용
  • 텍스트 일부를 강조할 때: span, strong, em 사용
  • 제목이 필요할 때: h1~h6 사용 (div 사용 금지)
  • 목록이 필요할 때: ul, ol, li 사용
  • 필요시 CSS의 display 속성으로 속성 변경
  • 의미있는(semantic) HTML 구조 작성
